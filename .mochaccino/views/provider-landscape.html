// NOTES
Consumer adapters vs Arcana providers
OpenAIEmbeddingProvider and ChromaDBVectorStore live in KyberBot's repo — they're not Arcana packages. They implement Arcana interfaces but are consumer-side wiring. The distinction matters: Arcana can't break them with an interface change unless it's additive.

KyberBot's ChromaDB adapter is being replaced by arcana-provider-sqlite-vec — no Docker dependency, same :memory: test pattern as libsql.
Testkit as specification
Every FakeXxx in arcana-testkit is both a test double and a living specification: it shows exactly what a real provider must implement. The compliance harness in runComplianceSuite is the formal gate — any real provider must pass it before shipping.

arcana-provider-sqlite-vec passes its own 11-test suite; compliance harness integration is T17.
Next provider: postgres
arcana-provider-postgres is the Brain migration gate. Brain runs on Convex (a serverless cloud DB). Migration shape: shed 5 Convex artifacts (4-flat-ID edges, shadow entity rows, _creationTime, reactive queries, loose enum vocab) → swap to arcana-provider-postgres.

Not started. Depends on KyberBot merge first.
