// NOTES
Publish order
arcana-contracts must publish first — all other packages reference it. The other 4 publish in any order. Previous v0.1.0 used npm publish leaving workspace:* in manifests. v0.1.1 fixed with pnpm publish -r.
arcana-config absorbed
A 5th package arcana-config existed briefly. Merged into arcana-core (src/config/) on 2026-05-19 — it had no external consumers and only added zod transitively. Simplification with no API change.
Provider gaps
StructuredStore: libsql (real) + testkit (fake). VectorStore: sqlite-vec (real) + testkit (fake). KyberBot's ChromaDB is a consumer adapter, not an Arcana package. arcana-provider-postgres is next — Brain migration gate.
