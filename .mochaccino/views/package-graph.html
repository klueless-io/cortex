Why publish order matters
arcana-contracts must publish first — all other packages reference it. The other 4 can publish in any order. Previous v0.1.0 publish used npm publish which left workspace:* in manifests instead of real semver. v0.1.1 published with pnpm publish -r — confirmed clean.
arcana-config disappearance
There was a 5th package: arcana-config. It was merged into arcana-core on 2026-05-19 (src/config/). It had no external consumers and only added zod as a transitive dep. Removing it simplified the graph without changing any public API.
Provider landscape gaps
StructuredStore: one real provider (libsql), one fake (testkit). VectorStore: one real provider (sqlite-vec), one fake (testkit). KyberBot's ChromaDB wrapper is a consumer adapter, not an Arcana package. arcana-provider-postgres is the next planned provider — needed for Brain migration.
